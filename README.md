# P2P VPN для Android (аналог Radmin VPN)

Приложение для создания виртуальной локальной сети между Android-устройствами.
Устройства подключаются друг к другу напрямую (P2P) через WebRTC.

## Архитектура

```
┌─────────────────┐         ┌─────────────────┐
│  Устройство A   │         │  Устройство B   │
│                 │         │                 │
│  ┌───────────┐  │  WebRTC │  ┌───────────┐  │
│  │VpnService │  │◄───────►│  │VpnService │  │
│  │  (TUN)    │  │  P2P    │  │  (TUN)    │  │
│  └─────┬─────┘  │DataChan.│  └─────┬─────┘  │
│        │        │         │        │        │
│  ┌─────┴─────┐  │         │  ┌─────┴─────┐  │
│  │   Apps    │  │         │  │   Apps    │  │
│  │10.0.0.1   │  │         │  │10.0.0.2   │  │
│  └───────────┘  │         │  └───────────┘  │
└─────────────────┘         └─────────────────┘
         │                           │
         └───── Firebase (только для обнаружения друг друга)
```

## Как работает

1. **Устройство A** создаёт "сеть" → получает 6-символьный код
2. **Устройство B** вводит этот код → присоединяется
3. Firebase используется ТОЛЬКО для начального обмена адресами (signaling)
4. После подключения трафик идёт НАПРЯМУЮ между устройствами (P2P)
5. VpnService создаёт TUN-интерфейс → IP-пакеты передаются через WebRTC DataChannel

## Два режима работы

### Режим 1: Firebase (рекомендуемый)
- Автоматическое обнаружение пиров
- Нужен аккаунт Firebase (бесплатный Spark план)
- UX как в Radmin VPN

### Режим 2: Ручной обмен ключами
- Без каких-либо серверов вообще
- Пользователи обмениваются Base64-ключами через мессенджер
- Работает даже без интернета (в одной WiFi сети)

## Настройка Firebase

1. Создайте проект в [Firebase Console](https://console.firebase.google.com)
2. Добавьте Android-приложение с package name: `com.radminvpn.android`
3. Скачайте `google-services.json` и положите в `app/`
4. Включите Realtime Database в Firebase Console
5. Установите правила безопасности:

```json
{
  "rules": {
    "networks": {
      ".read": true,
      ".write": true
    }
  }
}
```

> ⚠️ Для продакшена используйте более строгие правила с аутентификацией!

```

├── model/          # Модели данных

└── ui/             # UI (Activity)
```

## Технологии

- **Kotlin** + Coroutines + Flow
- **Android VpnService** — TUN интерфейс
- **WebRTC** (stream-webrtc-android) — P2P DataChannel
- **Firebase Realtime Database** — signaling (бесплатно)
- **Google STUN серверы** — NAT traversal (бесплатно)

## Ограничения

- Работает только L3 (IP-уровень), не L2 (Ethernet)
- Android позволяет только один VPN одновременно
- За жёстким CGNAT (некоторые мобильные операторы) P2P может не установиться
- Для LAN-игр с broadcast/mDNS нужны дополнительные workaround'ы
